CTB Supplier — User Guide

Cooking the Books — Supplier & Stock Item Management Portal

Document version 1.0  |  April 2026

Overview

The CTB Supplier Portal is a web-based application for managing supplier profiles and their associated stock items. It provides a single place to:

  • Maintain a catalogue of suppliers, including contact details, ABN, category, and website.
  • Manage stock items (products) for each supplier — pricing, units, categories, and media images.
  • Control which users can access the system and which suppliers they are permitted to view.
  • Generate API keys so that external systems (e.g. mobile apps, partner portals) can read supplier data programmatically.
  • Validate the health of product image URLs in bulk.

Getting access

Two-step access requirement Access to CTB Supplier requires both of the following conditions to be met. Satisfying only one is not sufficient to sign in.

Step 1 — Microsoft Entra authorisation

Your organisation's IT administrator must have added your email address to the Microsoft Entra ID (formerly Azure Active Directory) tenant used by this application. This gives you a valid organisational identity that the system can authenticate.

If you can sign in to other Microsoft 365 or Azure services with your @cookingthebooks.com.au account, Step 1 is already satisfied.

Step 2 — Registration in the CTB Supplier User List

Even after Microsoft Entra authentication succeeds, the application checks that your email address exists in its own internal User List and that your account is marked Active. If your email is not found, or your account has been deactivated, you will be shown an access-denied page and will not be able to proceed.

A system administrator must add your email address to the User List (see User management) before you can use the application.

What to do if you cannot sign in Contact your CTB Supplier administrator and ask them to:
  1. Confirm your Microsoft 365 account exists in the organisation's Entra tenant.
  2. Add your email address to the CTB Supplier User List and ensure Active is checked.

Security and what you can see

Once you have access, what you can see and what you can do are both governed inside this application:

ControlWhat it governsManaged by
Supplier restrictions Which suppliers a user is permitted to view and work with. A user with no restrictions can see all suppliers. A user with restrictions can only see the specific suppliers that have been assigned to them. Administrator via Admin → Users → Suppliers
Permissions What actions a user may perform — for example, whether they can access the Admin section to manage other users, permissions, and API keys. Administrator via Admin → Users → Permissions

Signing in

  1. Navigate to the CTB Supplier Portal URL provided by your administrator.
  2. Click the Sign in button in the top-right corner of the page.
  3. You will be redirected to the Microsoft sign-in page. Enter your organisational email and password (or use single sign-on if already signed in to Microsoft 365).
  4. Upon successful authentication, you are returned to the CTB Supplier Portal and signed in automatically.

To sign out, click your name in the top-right corner and select Sign out.

Session note Your session is tied to your Microsoft Entra token. If your IT administrator disables or removes your Entra account, or if a CTB Supplier administrator deactivates your User List entry, you will lose access on your next sign-in attempt.

Supplier details

The Supplier Details page displays a supplier's full profile together with all of their stock items.

Filtering stock items

Three filters can be applied simultaneously — all active filters are combined (AND logic):

FilterHow it works
BrandSelect from a dropdown of all brands recorded for this supplier. Choose (No Brand) to show items with no brand assigned.
CategorySelect from a dropdown of all categories recorded for this supplier.
DescriptionType any word or phrase to search within the stock item description (case-insensitive contains search).

Click Filter to apply. A Clear button appears whenever any filter is active — clicking it removes all filters and returns to the full list.

Filter values are preserved when you change the page size, switch between List and Grid view, or navigate between pages.

View modes

Toggle between List view (a detailed table) and Grid view (product cards with images) using the buttons at the top-right of the stock items section. Your preference is remembered in a browser cookie for 12 months.

Pagination

Use the Per page selector to show 25, 50, or 100 items at a time. Navigate between pages using the numbered pagination controls at the bottom of the page.

Actions

  • Edit Supplier — modify the supplier's name, ABN, website, description, category, or active status.
  • + New Stock Item — add a new product to this supplier.
  • Validate URLs — run a health check on all product image URLs (see URL validation).

Stock items

Stock items are the individual products that belong to a supplier. They can be accessed either from the Supplier Details page or from the global Stock Items section (which shows items across all suppliers you can access).

Editing a stock item

From the Supplier Details page, click the Stock Code link in list view (or the card header in grid view) to open the Edit screen for that item.

Deleting a stock item

From the global Stock Items list, use the Delete button on the row. A confirmation screen will be shown before the deletion is committed. Deletion is permanent and cannot be undone.

URL validation

The Validate URLs tool (accessible via the button on any Supplier Details page) performs a bulk health check on every product image URL recorded for that supplier.

How it works

When you click Validate URLs, the system sends an HTTP HEAD request to each recorded Media URL — up to 10 requests simultaneously — and records the result. The page is returned once all checks are complete.

Please wait For suppliers with a large number of stock items, the check may take several seconds to complete. Do not navigate away while the page is loading.

Reading the results

Four summary tiles are shown at the top of the page:

TileMeaning
URLs checkedTotal number of items that had a Media URL and were tested.
PassedURLs that returned an HTTP 2xx (success) response.
FailedURLs that returned a non-success HTTP status or could not be reached at all.
No URL setItems that have no Media URL recorded (shown only when at least one item has no URL).

If any URLs failed, a table is shown listing each failure with:

  • The Stock Code (linked directly to the Edit screen for that item).
  • The item's Description.
  • The URL that was tested (opens in a new tab when clicked).
  • The Result — either an HTTP status badge (e.g. HTTP 404) or a Error badge for network-level failures such as DNS errors or timeouts.
  • An Edit button to jump straight to the stock item and correct the URL.

If all URLs passed, a green success message is displayed instead of the table.

Note on HTTP 405 responses Some image hosting servers do not support HEAD requests and return HTTP 405 Method Not Allowed. This does not necessarily mean the image is missing — you can click the URL link to verify it loads in your browser.

User management

Admin only The Admin menu is only visible to users who have been granted the Add/Update AppUsers permission.

Navigate to Admin → Users to manage the list of people who are permitted to use the CTB Supplier Portal.

Adding a new user

  1. Click + New User.
  2. Enter the user's Full Name and Email Address. The email address must exactly match the address registered in Microsoft Entra.
  3. Ensure Active is checked.
  4. Click Save.
  5. Optionally, assign permissions and supplier restrictions to the new user.
Important Adding a user here does not create their Microsoft account. The user must already exist in your organisation's Microsoft Entra tenant. This step only grants them access to the CTB Supplier Portal after they authenticate via Microsoft.

Deactivating a user

To temporarily revoke access without deleting the account, click Edit next to the user, uncheck Active, and save. The user will be shown an access-denied page on their next sign-in attempt.

Deleting a user

Click Delete next to the user and confirm. This permanently removes the user record along with all of their associated permissions and supplier restrictions. This action cannot be undone.

Permissions

Permissions control what actions a user is allowed to perform within the application. They are assigned per-user from Admin → Users → Permissions (the Permissions button on any user row).

Available permission

PermissionWhat it grants
Add/Update AppUsers Access to the entire Admin menu, including: user management, permission definitions, login history, and API key management. Users without this permission cannot see or access the Admin section at all.

Granting or revoking a permission

  1. Go to Admin → Users.
  2. Click Permissions on the row of the user to modify.
  3. Check or uncheck the desired permission.
  4. Click Save.

Managing the permissions list

The set of available permissions itself is managed under Admin → Permissions. New permission types can be added and existing ones renamed or deleted here. Deleting a permission type removes it from all users who held it.

Supplier restrictions

By default, a user with no restrictions can see and work with all suppliers. Supplier restrictions allow you to limit a user to a specific subset of suppliers.

This is useful when, for example, an external user or a team member should only be able to access data for the suppliers they are responsible for.

Applying restrictions to a user

  1. Go to Admin → Users.
  2. Click Suppliers on the row of the user to restrict.
  3. Check the boxes next to each supplier this user is permitted to access.
  4. Click Save.
Leave all unticked to grant full access If no suppliers are checked, the user is treated as unrestricted and can access all suppliers. Only check boxes to limit access to specific suppliers.

Effect of restrictions

  • The Supplier list shows only the permitted suppliers.
  • Restricted users cannot create new suppliers.
  • Attempts to directly navigate to a restricted supplier's URL will be blocked with a Forbidden page.
  • The global Stock Items list is also filtered to show only items belonging to permitted suppliers.

API keys

API keys allow external systems to read supplier data programmatically via the CTB Supplier REST API. They are managed under Admin → API Keys.

The key is shown only once
When a new API key is generated, the full key value is displayed only on the creation screen. Only a one-way hash is stored in the database. If the key is lost, it cannot be recovered — generate a new one.

Generating a new key

  1. Click + Generate New Key.
  2. Enter a descriptive Key Name that identifies the consumer (e.g. Mobile App, Partner Portal).
  3. Click Generate.
  4. On the next screen, copy the displayed key immediately and store it securely. The key will not be shown again.

Using an API key

Include the key in every API request as the X-API-Key HTTP header: 

GET /api/v1/suppliers
X-API-Key: CTB_your_key_here

Refer to the interactive API documentation at /swagger for full endpoint details.

Revoking or deleting a key

  • Revoke — disables the key without removing it. It can be re-enabled later. Use this for temporary suspension.
  • Delete — permanently removes the key. Any system using it will immediately lose API access.

Frequently asked questions

I can sign in to Microsoft 365 but not to CTB Supplier. What should I do?

Your Microsoft account is valid, but you have not yet been added to the CTB Supplier User List. Contact your administrator and ask them to add your email address under Admin → Users.

I used to have access but now I see an "Inactive" message.

Your account has been deactivated in the User List. Contact your administrator to have your account re-enabled.

Why can I only see some suppliers and not others?

Your account has supplier restrictions applied. You will only see the suppliers that have been assigned to you. Contact your administrator if you need access to additional suppliers.

The Validate URLs page is taking a long time. Is that normal?

Yes. The check sends a real HTTP request to every product image URL. For suppliers with many stock items this can take 10–20 seconds or more. Leave the page open until it finishes loading.

A URL shows HTTP 405 in the validation results. Does that mean the image is broken?

Not necessarily. HTTP 405 means the server does not support the HEAD request method used by the validator. Click the URL link on the results page to open it in a new tab and verify whether the image actually loads.

I generated an API key but forgot to copy it. Can I retrieve it?

No. Only a one-way cryptographic hash of the key is stored. The plaintext key is displayed only once at creation. You will need to generate a new key and update any systems that were using the old one.

Where can I find the REST API documentation?

The interactive Swagger documentation is available at /swagger on the same host as the portal. All API requests require a valid API key in the X-API-Key header.
